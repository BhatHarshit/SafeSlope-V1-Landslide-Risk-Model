# Contributing to SafeSlope

Thank you for your interest in contributing to SafeSlope.

## Contribution Guidelines
- Follow existing code structure and naming conventions
- Write clean, well-documented code
- Clearly describe changes in pull requests
- Ensure experiments are reproducible

## Areas of Contribution
- Model optimization
- Feature engineering
- Dataset expansion
- Visualization and reporting
- Documentation improvements

## Workflow
1. Create a new branch for your work
2. Commit changes with clear messages
3. Open a pull request for review

This is a research-oriented project, so clarity and reproducibility are prioritized.
